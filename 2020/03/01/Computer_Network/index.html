计算机网络

框架：Y轴逻辑结构模型+X轴协议体系

一、概述

  • 简介：
    1. 概念：
      计算机网络是通过通信设备与线路将一个个分散的、具有独立功能的计算机系统连接起来，并由功能完善的软件实现资源共享和数据通信/信息传递的系统。
    2. 组成元素：
      • 独立的计算机系统：例如电脑、手机
      • 通信设备：例如路由器、交换机
      • 线路：例如网线、电缆
      • 软件：例如微信、QQ
      • 协议：TCP、IP等
    3. 边缘部分以及核心部分的分工 ：
      • 边缘部分：用户通过C/S方式或者P2P的方式直接进行资源共享和数据通信
      • 核心部分：为边缘部分提供资源共享等服务
    4. 分类：
      • 按分布范围分类：广域网WAN、城域网MAN、局域网WAN、个人区域网PAN
      • 按使用者分类：公用网、专用网
      • 按交换技术分类：电路交换、报文交换、分组交换
      • 按拓扑结构分类：总线型、星型、环型、网状型(例如因特网)
      • 按传输技术分类：广播式网络、点对点网络
    5. 功能：
      • 主要功能：资源共享、数据通信。
      • 次要功能：分布式处理、提高可替代性/可靠性、负载均衡等。
    6. 两种主要功能的实现方式：
      • 资源子网：用于实现资源共享，由应用层、表示层、会话层组成。
      • 通信子网：用于实现数据通信，由网络层、链路层、物理层组成。
  • 协议：
    1. 协议的定义：为网络中的对等实体数据交换而建立的规则、标准，为每一层需要实现的功能服务。
    2. 协议的组成：
      • 语法：规定传输数据的格式
      • 语义：规定所要完成的功能/数据需要包含的元素
      • 同步：规定各种操作的顺序
  • 计算机网络的结构：
    1. 七层OSI参考模型：
      • 应用层：
        执行所有能与用户交互产生网络流量的程序，常用的服务有FTP、HTTP、SMTP等。
      • 表示层：
        处理在两个通信系统中交换信息的表示方式，即规定的语法以及语义，主要的功能有数据的格式转换、数据的加密解密、数据的压缩和恢复。
      • 会话层：
        为表示层的用户进程建立连接并在连接上有序的传输数据，也称作建立同步(SYN)，主要的协议有ADSP、ASP。(注：同步也叫有序，类似于操作系统中进程的同步)
      • 传输层：
        负责不同主机中相互对应的两个进程之间的通信，即端到端/两个用户端之间的通信。主要的功能有可靠传输(一个基于确认机制的过程)、差错控制、流量控制、复用分用，主要的协议有TCP、UDP。
      • 网络层：
        把分组(可以理解为数据)从源端传到目的端，并为分组交换网上的不同主机提供通信服务，网络层的传输单位是数据报(数据报由分组组成)。主要功能有路由选择(选择最佳路径)、流量控制、差错控制、拥塞控制，主要协议有IP、IPX、ICMP、IGMP、ARP、RARP、OSPF。
      • 数据链路层：
        把网络层传下来的数据报组装成帧，主要功能有定义帧的开始和结束、差错控制、流量控制、访问/接入控制，主要协议有SDLC、HDLC、PPP、STP。
      • 物理层：
        实现比特流在物理媒介上的透明传输(计网以及操作系统中的透明是指”看不见”，即不管数据是啥都执行传输)，主要功能有定义接口特性、定义传输模式(单工、半双工、双工)、定义传输速率、比特同步、比特编码，主要协议有Rj45、802.3。
    2. TCP/IP协议栈：
      • HTTP、FTP、DNS
      • TCP、UDP
      • IP
      • Ethernet、ATM、Frame Relay
    3. 根据协议栈产生的四层TCP/IP参考模型：
      • 应用层
      • 传输层
      • 网际层
      • 网络接口层
    4. 综合以上两种模型而得出的五层参考模型：
      • 即将应用层、表示层、会话层统称为应用层，下面四层不变。

二、应用层

  • 简介：对应用程序的通信提供服务。
  • 网络应用模型：
    1. 客户/服务器模型(C/S模型)：客户请求计算服务，服务器提供计算服务。
    2. P2P模型：每个主机既请求服务也提供服务，一种去中心化的思想。
  • 协议：
    1. FTP：基于C/S模型的文件传送协议，可以理解为拷贝(从FTP客户端拷贝到FTP服务器就叫上传，从FTP服务器拷贝到FTP客户端就叫下载)。
    2. SMTP、POP3：以发邮件为例，SMTP是发邮件使用的协议，POP3是收邮件使用的协议。发送者通过SMTP协议将邮件发给发送端邮件服务器，发送端邮件服务器再通过SMTP协议将邮件发给接收端邮件服务器，接收端邮件服务器通过POP3协议将邮件发给接收者。
    3. HTTP：一种超文本传输协议，用户与万维网服务器建立连接后，万维网服务器通过HTTP协议将用户请求的指定URL(统一资源定位符)资源发送给用户。
    4. DNS：将要访问网站的域名转换成其服务器的IP地址，解析方式分为递归和迭代。
  • 功能：文件传输+访问+管理(FTP)、电子邮件(SMTP、POP3)、虚拟终端(HTTP)、远程登录(DNS)。
  • DHCP协议：自动给DHCP服务器范围内的设备分配IP。

三、传输层

  • 简介：通过端口提供进程与进程之间的逻辑通信。
  • UDP协议：无连接的不可靠的用户协议，即不需要建立连接收到报文后也不需要给出确认，适用于小文件。
  • TCP协议：
    1. 定义：面向连接的可靠的传输控制协议，可靠的面向连接也就导致了有时延，适用于大文件之间的传输。
    2. 结构：见教材
    3. 连接方式：三次握手四次挥手
    4. 功能：可靠传输、流量控制(滑动窗口实现)、拥塞控制。

四、网络层

  • 简介：
    1. 主要任务：把分组从源端传到目的端，为分组交换网上的不同主机提供通信服务。
    2. 传输单位：数据报–由多个分组构成。
    3. 功能：路由选择、异构网络互联、拥塞控制(与点对点的流量控制相比，拥塞控制是全局的控制)。
  • 数据交换方式：
    1. 电路交换：类比打电话
      • 建立连接
      • 通信
      • 释放连接
    2. 报文交换：信息源以报文的形式经过交换设备的短暂存储等待找到可用的空闲链路进行转发后到达接收发。
    3. 分组交换：就是将报文交换中的报文进行分组，即分割成小的数据块进行交换。
  • IP数据报：
    1. 组成：首部(固定部分和可变部分)、数据(传输层的报文段)
    2. 分片：由于链路层能接收的数据帧(网络层的数据报+首尾帧)有长度要求，故数据报的长度有上限，若数据报的长度超过上限，就要对其进行分片。
  • IP地址：
    1. 简介：
      • MAC地址：MAC地址是全球唯一且只要网卡/适配器不变，该地址就不变是设备(如电脑)的身份证。
      • IP地址：IP地址是全球唯一(but随着所连接网络区域的改变而改变)的32位/4字节标识符，标识路由器主机的接口。
    2. 作用：提供IP地址通俗讲就是网络区域+区域内的主机号，以方便发送方迅速找到接收方，而具体的IP地址是根据不同的IP编址方式决定的。
    3. IP(IPv4)编址方式：
      • 根据分类编址：网络号+主机号
        主要有A、B、C三类，先用二进制计算，最后转为十进制，具体的编码方式自己可推。
      • 根据子网划分编址：外部IP地址+子网掩码+主机号，或者网络号+子网号+主机号
        其中对IP地址和子网掩码(子网掩码中的主机号全为零)做二进制的与运算可得到子网网络地址，子网网络地址加上主机号即最终的目的地址，最后将8bit一组的二进制结果转化为十进制。
      • 无分类编址CIDR：网络前缀+主机号
        在IP地址后加上”/“，然后写上网络前缀的位数，网络前缀相同的IP地址组成一个CIDR地址块。
      • 构成超网：
        通过将网络前缀缩短使多个子网聚合成较大的子网。
  • 移动IP：
    1. 定义：即使移动了物理地址，也能使用某个原先固定的网络IP地址实现异地漫游，例如在家也能使用公司的IP地址访问公司的内部资源。
    2. 实现方式：将当前移动了的物理地址作为转交地址发给原网络IP地址，访问的资源会通过原网络IP地址发送到该转交地址上，从而实现漫游。
  • IP数据报传播的三种方式：
    1. 单播：点对点
    2. 广播：点对多点，发送数据报到同一广播域内的所有设备。
    3. 组播/多播：点对多点，但将数据报发送给指定的设备。
  • ARP协议：完成主机或路由器IP地址到MAC地址的映射。
  • IPv6：解决IPv4地址不够分配的问题，一般用冒号十六进制表示法。
  • 其他协议：RIP、OSPF、BGP等
  • 网络层设备：路由器–功能是转发分组。

五、数据链路层

  • 基本概念：
    1. 节点：主机、路由器
    2. 链路：物理(有线/无线)通道，链路的传输介质有光纤、双绞线、微波等。
    3. 数据链路：逻辑通道，由链路、实现协议的硬件和软件组成。
    4. 帧：链路层的协议数据单元，封装网络层数据报。
    5. 数据链路层的作用：通过链路在每条物理链路相邻的两个节点之间传输数据报。
    6. 无差错接收：接收方根据差错检测结果只接收对的帧，错的则丢弃。
    7. 可靠性传输：接收方接收传输来的每一个帧无论对错。
  • 链路层的功能：
    1. 接收下层/物理层提供的服务，并为上层/网络层提供服务：将来自网络层的数据可靠的传输到相邻节点的目标网络层。
    2. 链路管理：连接的建立、维持、释放。
    3. 封装成帧：在网络层提供的IP数据报的前面加上首部帧，在IP数据报的后面加上尾部帧。
    4. 流量控制：
      • 简介：根据接收的需求，控制发送方的流量。
      • 流量控制的方法：
      1. 停止-等待协议(也可以存在于传输层)：每发送一个帧就停止发送，等待对方返回确认的帧(ACK)，再继续发送下一个帧。
      2. 滑动窗口协议(GBN协议、SR协议)：
        • GBN协议(后退N帧协议)：
          发送窗口：依次发送窗口内的一个帧，等收到窗口下界(最左侧)的确认帧(ACK)向右滑动一个窗口，否则发送方不移动窗口。
          接收窗口：长度为1，收到窗内的帧就右移并发送确认帧，否则一直发送上一帧的确认帧，且不可以缓存接收其他帧。
        • SR协议(选择重传协议)：
          发送窗口：依次发送窗口内的一个帧，等收到窗口下界(最左侧)的确认帧向右滑动一个窗口，否则发送方不移动窗口，且循环发送窗口内未收到确认的帧。
          接收窗口：只要接收到下界的帧，窗口就右移并发送该帧的确认帧，且可以缓存接收窗口内的其他帧并发送这些帧的确认帧。
        • 小结：发送窗口收到下界帧的确认帧才右移，接收窗口收到下界帧就右移。
      3. 总结：
        • 停止等待协议相当于一个长度的发送窗口，一个长度的接收窗口。
        • GBN协议相当于多个长度的发送窗口，一个长度的接收窗口。
        • SR协议相当于多个长度的发送窗口，多个长度的接收窗口。
    5. 差错控制(帧错/位错)：
      • 位错：比特位出错，0变成1，1变成0。
      • 纠错编码：海明码–发现双比特错，纠正单比特错。
      • 帧错：帧缺少、帧重复、帧失序。
  • 链路层的两种信道：
    1. 广播信道：一对多，类似于广播形式，可以是有线介质连接，也可以是无线介质连接。
    2. 点对点信道：一对一。
  • 信道划分介质访问控制：
    1. 信道划分的目的：
      • 划分信道是为了在广播信道中实现介质访问控制，使信道/用户之间不会相互干扰，信道/用户之间不会发生冲突，例如对讲机不能多人同时说话。
    2. 划分方式(也叫分配/使用信道的方式)：
      • 静态划分：频分复用(FDM)、时分复用(TDM)、波分复用(WDM)、码分复用(CDM)。
      • 动态划分方式：
      1. 轮询协议：主结点轮流”邀请”子节点发送数据。
      2. 令牌传递协议：通过令牌来控制信道的使用，确保同一个时刻只有唯一拥有令牌的那个结点在持有令牌的时间内独占信道。
      3. ALOHA协议：不监听信道，不按时间槽发送，想发就发，超时未接收到则随机时间重发。
      4. CSMA协议：发送消息之前监听信道，看信道是否空闲，空闲则立即发送，忙则推迟发送。
      5. 等
  • 局域网与广域网：
    1. 局域网：
      • 定义：在一个区域内由多台计算机互联形成的计算机组，连接的方式是广播，故需要考虑信道划分以及相关协议。
      • 分类：以太网、令牌环网、FDDI网、ATM网、无线局域网(WLAN)。
    2. 广域网：
      • 定义：覆盖很大物理范围的国际性远程网络，它实现了分布在不同地区局域网之间的相互连接，连接的方式是点对点，例如因特网。
      • 协议：PPP协议、HDLC协议–这两个协议采用的都是全双工通信
      1. PPP协议三个组成部分：
        • 将网络层的IP数据报封装到串行链路。
        • 链路控制协议LCP：建立连接数据的物理链路。
        • 网络控制协议NCP：建立连接网络层协议的逻辑链路。
      2. HDLC协议：一个在同步网上传输数据、面向比特流的数据链路层协议。
  • 链路层的两个设备：
    1. 网桥：扩展以太网的设备，对链路层上中转过来的帧执行先过滤再转发的操作，提高了吞吐量。
    2. 交换机：相当于多接口网桥。

六、物理层

  • 数据通信基础：
    1. 基本术语：
      • 数据：二进制信息实体。
      • 信号：数据在传输过程中的存在形式，分为数字信号(类比基带信号)和模拟信号(类比宽带信号)。
      • 信源：发送数据源。
      • 信道：信号传输媒介
        1. 按信号类型分：数字信道、模拟信道
        2. 按传输介质分：有线信道、无线信道
      • 信宿：接收端。
      • 码元：用一个单位时间的不同的信号波形代表不同的离散数值，该基本波形就叫做码元。如一个零电平就代表一个码元，当码元为k进制时，一码元为log2(k)个bit。
      • 码元传输速率：单位时间数据传输系统传输码元的个数，单位为波特(Baud)/(码元/s)。
      • 信息传输速率：单位时间传输比特数目，单位为bit/s。
      • 带宽：最大信息传输速率，单位是bit/s。
      • 信道带宽：信道能通过的最高频率与最低频率之差，单位是Hz。
    2. 数据通信模型：
      • 发送端/信源：提供需要传输的数字信号，并发送给调制解调器。
      • 调制解调器/发送器：将接收到的数字信号转为模拟信号从而实现在电话线路上的传播，并通过电话线路传到公用电话网上。
      • 公用电话网/传输系统：作为中介将模拟信号传给指定的接收器。
      • 调制解调器/接收器：将通过公用电话网接收到的模拟信号转为数字信号，并将数字信号发送给接收端。
      • 接收端/信宿：接收数字信号，输出信息。
    3. 三种通信方式：
      • 单工通信：单方向通信，仅需一条信道。
      • 半双工通信：双向通信，但通信双方不能同时接收和发送信息，需要两条信道。
      • 全双工通信：双向通信，且通信双方能同时发送和接收信息，需要两条信道。
    4. 两种数据传输方式：
      • 串型传输：一条信道进行传输。
      • 并行传输：多条信道同时发送多个比特。
  • 奈氏准则和香农定理：
    1. 奈氏准则：
      • 该准则只针对码元传输速率，即单位是Baud，码元/s。
      • 准则内容：在理想低通(低于最高频率的信号都可以通过)条件下，为了避免码间串扰，最高码元传输速率为2W Baud，W为信道带宽。
    2. 香农定理：
      • 信噪比：信号的平均功率/噪声的平均功率，即为S/N。
      • 准则内容：在带宽受限且有噪声的信道中，信息的数据传输速率最高为Wlog2(1+S/N)，单位bit/s，W为信道带宽。
  • 编码与调制：
    1. 编码：
      • 定义：将数字数据(如文字数据)/模拟数据(如声音数据)转换成数字信号
      • 编码方式：非归零编码、曼彻斯特编码等
    2. 调制与解调：
      • 调制定义：将数字数据/模拟数据转换成模拟信号
      • 调制方式：调幅、调频、调相等
      • 解调：将模拟信号转换为计算机可读的数字信号
  • 传输介质：
    1. 定义：位于物理层的下一层(可以理解为第零层)，是数据传输系统中的物理通路。
    2. 分类：
      • 导向性传输介质：特定的传播方向，电缆、光纤等。
      • 非导向性传输介质：自由空间传播，空气中的无线电波、微波、激光等。
  • 两个物理层传输设备：
    1. 中继器：再生与还原信号以及增加信号传输距离的设备，用于避免由于信号在传输中的衰减造成的失真。
    2. 集线器：多接口的中继器。

七、总结

  • 结构体系：应用层、传输层、网络层、数据链路层、物理层，前两层为端到端，后三层为点到点。
  • 协议体系：
    1. 应用层
      • DNS协议、FTP协议、HTTP协议、HTTPS协议、SMTP协议等。
    2. 传输层
      • UDP协议：无连接，不可靠的特点
      • TCP协议：面向连接，可靠，连接控制机制(三次握手，四次挥手)、流量控制机制(滑动窗口、慢启动、拥塞避免、快速重传)等。
    3. 网络层
      • IP协议、ICMP协议、ARP协议等。
    4. 数据链路层
      • RARP协议、Ethernet协议、ATM协议等。
