Spring&Mybatis

一、Mybatis基础

  • 准备工作(在项目的基本目录pom.xml中导入相关jar包/依赖)：
    xml
  • Mybatis部分：
    1. 在pojo包中编写与数据库中表对应的实体类–如User.java：只有一个原则，该类的属性必须与对应数据库中表的列名一致。
    2. 在mapper包中编写与sql映射文件对应的映射接口–如UserMapper.java：略
    3. 在mapper包中编写sql映射文件–如UserMapper.xml：用来绑定接口
      xml
    4. 在resources中编写全局配置文件–如mybatis-config.xml：
      xml
  • 编写测试类：
    java
  • 注：关于报错以及纠错，原则就是在输出窗口依次从下往上纠错。

二、Spring整合Mybatis方式一

  • 准备工作(在项目的基本目录pom.xml中导入相关jar包/依赖)：略
  • Mybatis部分：
    1. pojo/实体包的实体类：同Mybatis基础
    2. mapper/映射包的映射接口类：同Mybatis基础
    3. mapper/映射包的xml映射文件：同Mybatis基础
    4. mybatis全局配置文件：省略了连接数据库以及将sql映射文件注入全局配置文件这两个部分。
  • Spring部分：即配置Spring.xml文件的四(3”固定”+1”可变”)要素以及映射接口的实现类
    1. 配置数据源：
      xml
    2. 配置SqlSessionFactory(固定写法)：
      xml
    3. 配置sqlSessionTemplate(固定写法)：
      xml
    4. 编写映射接口的实现类：
      java
    5. 配置sqlSession：最终就是通过该配置中的对象来操作数据库，对数据库进行增删改查等
      xml
  • 测试类：
    java

三、Spring整合Mybatis方式二

  • 准备工作：同方式一
  • Mybatis部分：同方式一
  • Spring部分：
    1. 配置数据源：同方式一
    2. 配置SqlSessionFactory：同方式一
    3. 跳过配置sqlSessionTemplate步骤，直接编写映射接口的实现类。且该实现类内部也不用通过set方法将注入的sqlSession对象变成我们要使用的SqlSessionTemplate对象，而是直接通过getSqlSession()方法得到一个sqlSession对象。
      java
    4. 配置sqlSession：
      xml
  • 测试类：同方式一
    java

四、配置事务

  • 事务定义：
    事务就是把一系列的动作当成一个独立的工作单元，这些动作要么全部完成，要么全部不起作用。
  • 配置事务的步骤：
    1. 在Spring的配置中导入约束：
      xml
    2. 配置事务通知：
      xml
    3. 导入aop/面向切面的头文件：
      xml
      END
