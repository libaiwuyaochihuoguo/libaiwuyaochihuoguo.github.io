操作系统

一.基本概念及原理

  • 层次结构：硬件之上，应用程序之下。
  • 主要功能：
    1. 从用户角度：操作系统是一个控制软件，用于管理应用程序，为应用程序提供服务。
    2. 从资源角度：管理资源、分配资源、管理外设。
    3. 整体而言，操作系统避免了应用程序直接访问底层硬件而造成的安全性问题，同时为上层的应用程序提供了简洁的底层硬件以及外设的接口，兼顾了安全和便捷。
  • 内部组件：CPU调度器(cpu进程管理)、物理内存以及虚拟内存管理(内存管理)、文件系统管理(磁盘管理)。
  • 特征：并发、共享、虚拟、异步(程序的执行，走走停停)

    二.中断、异常以及系统调用

  • 中断简介：
    1. 来源：来源于外设(不同硬件设备的计时器和网络的中断)
    2. 概念：通过中断使操作系统来感知到外设发出了请求，并对其进行处理。
    3. 处理时间：异步
    4. 处理响应的状态：持续，因为中断时操作系统会悄悄地完成接下来的工作，而应用程序不会感觉到中断。
  • 异常简介：
    1. 来源：来源于不良的应用程序或者应用程序意想不到的行为。
    2. 概念：不良的应用程序发出的非法指令或其他坏的处理状态(如内存出错)，使得操作系统不得不完成相应的功能。
    3. 处理时间：同步
    4. 处理响应的状态：操作系统会杀死该程序或者重新执行该意想不到的应用程序指令。
  • 系统调用简介：
    1. 来源：来源于应用程序
    2. 概念：是应用程序主动向操作系统发出的服务请求，并希望操作系统提供相应的支持。
    3. 处理时间：同步或异步(应用程序发出调用请求时，应用程序和操作系统是同步的，而操作系统返回调用时有可能与应用程序异步)
    4. 处理响应的状态：等待和持续
  • 中断的处理机制：
    1. 硬件需要做的处理：(中断的来源是外设以及硬件)
      • 将所有事件设置中断标记，一旦产生中断，将该标记提供给操作系统，操作系统根据标记也就知道了何处出现了问题。
    2. 操作系统需要做的处理：
      • 保存当前处理状态，即中断前一时刻的工作状态。
      • 根据中断标记，处理该中断程序。
      • 清楚中断标记。
      • 恢复之前的处理状态。
  • 异常的处理机制：(类似于硬件的中断标记，异常也有一个异常编号)
    1. 操作系统需要做的处理：
      • 保存现场
      • 异常处理(根据异常编号，杀死异常或者重新执行异常指令)
      • 恢复现场
  • 系统调用的处理机制：
    1. 应用程序进行系统调用时，是通过访问高层次的API接口而不是直接进行系统调用。

      三.内存管理

  • 基础：
    1. 内存的层次结构：
      • 从cpu寄存器到cache缓存、到主存、到磁盘，执行速度从快到慢。
    2. 操作系统需要分配的两块空间–逻辑空间与物理空间：
      • 逻辑地址空间：此空间是一个虚拟空间，给应用程序提供一个逻辑地址，使其在不接触底层硬件的情况下进行连续的工作。
      • 物理地址空间：分为主存和磁盘。
        主存：给优先级较高的正在运行的应用程序提供内存空间。
        磁盘：给优先级较低或者正在等待的应用程序提供内存空间。
    3. 地址空间：
      • 地址空间定义：
        • 逻辑地址空间：一个运行程序所拥有的内存范围
        • 物理内存空间：硬件支持的地址空间，主存以及磁盘。
      • 地址空间如何生成以及逻辑地址如何转换为物理地址：
        • 逻辑地址空间：程序的每行源码经过编译器等过程生成一个逻辑地址，最终整个程序会生成一段连续的逻辑地址。
        • 逻辑地址转换为物理地址的步骤：
          • 操作系统提供逻辑地址和物理地址之间的映射
          • cpu中的存储管理单元(MMU)以RAM为桥梁根据映射找到逻辑地址对应的物理地址
      • 地址安全检查(操作系统如何保护地址空间)：
        • 操作系统根据逻辑地址的基址和界限(相当于起始地址和长度)确认该地址是否合法，若不合法则会产生OutOfMemory异常，操作系统对其进行异常处理。
  • 内存分配：
    1. 连续内存分配策略：(连续是指整个程序的连续)
      • 首次适配：从地址为零的地方开始，将第一块满足程序空间使用要求的可用空闲块分配给该程序。
      • 最优适配：将最小的满足程序空间使用要求的可用空闲块分配给该程序。
      • 最差适配：将最大的可用空闲块分配给该程序。
      • 以上三种内存分配方法都会产生内存碎片，还需要进行碎片整理(压缩式碎片整理、交换式碎片整理)，以提高资源的利用率。
    2. 非连续内存分配策略：
      • 分段：将逻辑地址连续的程序按照一定规则(比如按照堆、栈、共享数据)分段，按照映射将这些段不连续的插入到物理地址中，可以避免连续时产生的内存碎片问题。
