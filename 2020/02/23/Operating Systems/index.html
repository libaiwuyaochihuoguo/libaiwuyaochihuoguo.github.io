操作系统

一.基本概念及原理

  • 简介：
    1. 定位：硬件之上，应用程序之下
    2. 从用户角度(从中间往上看)：操作系统是一个控制软件，提供一系列接口来管理应用程序，为应用程序提供服务。
    3. 从资源角度(从中间往下看)：管理进程(cpu)、管理内存(RAM)、管理文件(磁盘)、管理I/O(外设)。
    4. 整体而言，操作系统避免了应用程序直接访问底层硬件而造成的安全性问题，同时为上层的应用程序提供了简洁的底层硬件以及外设的接口，兼顾了安全和便捷。
  • 四种主要功能：
    1. 进程管理：cpu合理的调度进程/程序的执行。
    2. 内存管理：所有需要运行的进程/程序都需要放入内存(RAM)中，才能被cpu处理。
    3. 文件管理：合理的管理磁盘文件。
    4. 设备管理(I/0管理)：合理的分配外设给需要的进程，并在使用后进行回收。
  • 四种特征：
    1. 并发：一段时间内多个程序/进程同时运行。
    2. 共享：共享资源(互斥共享和同时共享)，分别以摄像头和扬声器为例。
    3. 虚拟：将物理实体变为若干个逻辑对应物，如虚拟内存技术可以使用户感觉运行时的内存远远超过实际的物理内存/RAM。
    4. 异步：由于资源有限，程序/进程的执行是走走停停而不是一贯到底。
  • 运行机制：
    1. 两种指令：特权指令与非特权指令
    2. 两种处理器状态：核心态与用户态
    3. 两种程序：内核程序与应用程序
  • 操作系统的结构：
    1. 内核：(必不可少的功能)
      • 时钟管理、中断处理、设备驱动、cpu切换、原语(原子操作，通过关中断和开中断来保证在开中断之前程序不可以被打断)
      • 四种主要功能(进程管理、内存管理、文件管理、设备管理)
    2. 非内核：
      • 非内核功能

        二.OS框架

  • 由中向上(由OS到应用程序和用户)：
    1. 作用：向用户提供服务
    2. 实现方式：
      • 命令接口：终端输入指令
      • 程序接口：系统调用
      • GUI：图形界面
  • 由中向下(由OS到硬件)：
    1. 作用：更好的管理资源
    2. 实现方式：
      • 进程管理
      • 内存管理
      • 文件管理
      • 设备管理
      • 等

        三.中断、异常以及系统调用

  • 中断(外中断)简介：
    1. 来源：来源于外设(cpu外部，如不同硬件设备的计时器和网络的中断)
    2. 概念：通过中断使操作系统来感知到外设发出了请求，并对其进行处理，cpu的状态从用户态转为核心态。(注：内外中断是用户态切换为核心态的唯一途径)
    3. 处理时间：异步
    4. 处理响应的状态：持续，因为中断时操作系统会悄悄地完成接下来的工作，而应用程序不会感觉到中断。
  • 异常(内中断)简介：
    1. 来源：来源于不良的应用程序或者应用程序意想不到的行为(cpu内部)
    2. 概念：不良的应用程序发出的非法指令或其他坏的处理状态(如内存出错)，使得操作系统不得不完成相应的功能。
    3. 处理时间：同步
    4. 处理响应的状态：操作系统会杀死该程序或者重新执行该意想不到的应用程序指令。
  • 系统调用(内中断)简介：
    1. 来源：来源于应用程序内部
    2. 概念：是应用程序主动向操作系统发出的服务请求，并希望操作系统提供相应的支持，只要是与资源有关的操作都需要进行系统调用来保证系统的安全性。
    3. 处理时间：同步或异步(应用程序发出调用请求时，应用程序和操作系统是同步的，而操作系统返回调用时有可能与应用程序异步)
    4. 处理响应的状态：等待和持续
  • 中断的处理机制：
    1. 硬件需要做的处理：(中断的来源是外设以及硬件)
      • 将所有事件设置中断标记，一旦产生中断，将该标记提供给操作系统，操作系统根据标记也就知道了何处出现了问题。
    2. 操作系统需要做的处理：
      • 保存当前处理状态，即中断前一时刻的工作状态。
      • 根据中断标记，处理该中断程序。
      • 清楚中断标记。
      • 恢复之前的处理状态。
  • 异常的处理机制：(类似于硬件的中断标记，异常也有一个异常编号)
    1. 操作系统需要做的处理：
      • 保存现场
      • 异常处理(根据异常编号，杀死异常或者重新执行异常指令)
      • 恢复现场
  • 系统调用的处理机制：
    1. 应用程序进行系统调用时，是通过访问高层次的API接口而不是直接进行系统调用，操作系统再根据具体的调用对接硬件资源。

      四.进程管理

  • 定义：
    1. 进程实体：由PCB(process control block–进程存在的唯一方式，保存了进程的状态等信息，用于恢复进程)、程序段、数据段组成，运行时在RAM中产生。
    2. 进程：进程是进程实体的一次运行过程，是资源分配的一个基本单位。
  • 组织方式：
    1. 链接方式：按照进程状态将PCB存储为队列形式，操作系统持有指向队列的指针。
    2. 索引方式：按照进程的状态建立索引表，操作系统持有指向索引表的指针。
  • 特征：
    1. 动态性：产生到消亡
    2. 并发性：可并发执行
    3. 独立性：独立运行
    4. 异步性：走走停停
    5. 结构性：每个进程都有一个PCB、程序段、数据段
  • 进程的状态以及状态转换：
    1. 状态：
      • 创建状态：操作系统为进程分配内存空间等资源，并为其创建以及初始化PCB的状态。
      • 就绪状态：已经拥有来运行所需要的资源，等待cpu空闲即可运行的状态。
      • 运行状态：占有cpu，并且正在cpu上运行的状态。
      • 阻塞状态：等待某个资源或者等待某个事件因而暂时不能运行的状态。
      • 终止状态：进程结束或者撤销并对资源进行回收的状态。
      • 挂起状态：引入虚拟内存技术，将暂时不能运行的进程调至外存等待，但该进程的PCB不会调到外存。
    2. 状态之间的转换：
      • 上述状态之间的转换。
  • 进程通信：
    1. 定义：进程之间的信息交换
    2. 方式：共享存储、管道通信、消息传递
  • 线程：
    1. 简介：程序执行流和cpu调度的最小单位，共享进程的资源。
    2. 属性：线程ID、TCB(Thread control block)、执行状态
    3. 实现方式：
      • 用户级线程：线程由应用程序通过线程库实现，cpu在用户态下即可完成，即用户能看到的线程。
      • 内核级线程：在核心态下完成的线程，即操作系统内核能看见的线程，内核级线程才是cpu分配的单位。
  • 进程调度的方式：
    1. 非抢占式调度：只允许进程主动放弃cpu。
    2. 抢占式调度：可以强制进程放弃cpu。
  • 调度算法：
    1. 先来先服务(FCFS)：先来先被调度
    2. 最短时间优先(SJF/SPF)：当前已到达且运行时间最短的先被调度
    3. 最高响应比优先(HRRN)：当前响应比最高的先被调度，其中响应比为：(等待时间+要求运行时间)/要求运行时间
    4. 时间片轮转调度算法(RR)：按照进程到达就绪队列顺序，轮流分配一个时间片，若进程在时间片内未完成，则剥夺cpu重新排队。
    5. 优先级调度算法：当前优先级最高的进程先被调度
    6. 多级反馈队列调度算法：
      • 是以上算法的平衡
        先设置多级优先队列，优先级从高到低，时间片从小到大。
        新进程到达时进入优先级最高的第一级队列，按FCFS原则等待分配时间片。
        时间片内未结束，则该进程进入下一个优先级队列。
        只有第k级队列为空时，才为第k+1级队列分配时间片。
  • 进程同步以及进程互斥：
    1. 进程同步：亦称直接制约关系，指某些产生直接合作的进程为了完成某种任务协调他们的工作次序而产生的制约关系。
    2. 进程互斥：亦称间接制约关系，指一个进程访问某个临界资源(比如打印机)时，另一个进程必须等待。
    3. 用来实现同步和互斥的信号量机制(P操作和V操作)以及进程互斥算法：(见教材)

      五.内存管理

  • 基础：
    1. 内存的层次结构：
      • 从cpu寄存器到cache缓存、到主存、到磁盘，执行速度从快到慢。
    2. 操作系统需要分配的两块空间–逻辑空间与物理空间：
      • 逻辑地址空间：此空间是一个虚拟空间，给应用程序提供一个逻辑地址，使其在不接触底层硬件的情况下进行连续的工作。
      • 物理地址空间：分为主存和磁盘。
        主存：给优先级较高的正在运行的应用程序提供内存空间。
        磁盘：给优先级较低或者正在等待的应用程序提供内存空间。
    3. 地址空间：
      • 地址空间定义：
        • 逻辑地址空间：一个运行程序所拥有的内存范围
        • 物理内存空间：硬件支持的地址空间，主存以及磁盘。
      • 地址空间如何生成以及逻辑地址如何转换为物理地址：
        • 逻辑地址空间：程序的每行源码经过编译器等过程生成一个逻辑地址，最终整个程序会生成一段连续的逻辑地址。
        • 逻辑地址转换为物理地址的步骤：
          • 操作系统提供逻辑地址和物理地址之间的映射
          • cpu中的存储管理单元(MMU)以RAM为桥梁根据映射找到逻辑地址对应的物理地址
      • 地址安全检查(操作系统如何保护地址空间)：
        • 操作系统根据逻辑地址的基址和界限(相当于起始地址和长度)确认该地址是否合法，若不合法则会产生OutOfMemory异常，操作系统对其进行异常处理。
  • 内存分配：
    1. 连续内存分配策略：(连续是指整个程序的连续)
      • 首次适配：从地址为零的地方开始，将第一块满足程序空间使用要求的可用空闲块分配给该程序。
      • 最优适配：将最小的满足程序空间使用要求的可用空闲块分配给该程序。
      • 最差适配：将最大的可用空闲块分配给该程序。
      • 以上三种内存分配方法都会产生内存碎片，还需要进行碎片整理(压缩式碎片整理、交换式碎片整理)，以提高资源的利用率。
    2. 非连续内存分配策略：
      • 分段：将逻辑地址连续的程序按照一定规则(比如按照堆、栈、共享数据)分段，按照映射将这些段不连续的插入到物理地址中，可以避免连续时产生的内存碎片问题。
      • 分页：与分段类似，区别就是分段地址偏移是可变的，分页的页帧的大小是固定的。
        TLB：cpu中的快表，用于将常用的页表项(可以看成存储逻辑地址和物理地址映射的一个数组)缓存到cpu中，缩短寻址时间。
        分页地址空间：见教材
        页寻址方案：见教材
    3. 实现虚拟内存的三种技术：
      • 覆盖技术：(在较小的可用内存中运行较大的程序，发生在程序之内)
        按照程序自身的逻辑结构，将程序分成若干个独立的程序模块，那些不会同时运行的模块共享同一块内存空间，这些程序模块按时间顺序执行。必要或常用的模块放入内存，不常用的放入外存。不存在调用关系的模块不必同时存入内存，从而完成了覆盖，且共享了同一块内存。
        总结：就是将RAM按程序的逻辑结构分层/分区(可以将其看成树结构)，分区原则就是每块区域中可能存放的程序模块不会同时运行/相互独立。
      • 交换技术：(发生在程序之间)
        操作系统将需要运行的程序导入到内存，将不需要运行的程序导出到外存/硬盘(进程挂起)。(单位是程序而不是模块，区别于覆盖技术)
      • 虚存技术(虚拟内存管理技术)：
        程序之内既采用覆盖技术也采用部分交换技术，整体而言是覆盖和交换技术的耦合与优化。
