操作系统

一.基本概念及原理

  • 简介：
    1. 定位：硬件之上，应用程序之下
    2. 从用户角度(从中间往上看)：操作系统是一个控制软件，提供一系列接口来管理应用程序，为应用程序提供服务。
    3. 从资源角度(从中间往下看)：管理进程(cpu)、管理内存(RAM)、管理文件(磁盘)、管理I/O(外设)。
    4. 整体而言，操作系统避免了应用程序直接访问底层硬件而造成的安全性问题，同时为上层的应用程序提供了简洁的底层硬件以及外设的接口，兼顾了安全和便捷。
  • 四种主要功能：
    1. 进程管理：cpu合理的调度进程/程序的执行。
    2. 内存管理：所有需要运行的进程/程序都需要放入内存(RAM)中，才能被cpu处理。
    3. 文件管理：合理的管理磁盘文件。
    4. 设备管理(I/0管理)：合理的分配外设给需要的进程，并在使用后进行回收。
  • 四种特征：
    1. 并发：一段时间内多个程序/进程同时运行。
    2. 共享：共享资源(互斥共享和同时共享)，分别以摄像头和扬声器为例。
    3. 虚拟：将物理实体变为若干个逻辑对应物，如虚拟内存技术可以使用户感觉运行时的内存远远超过实际的物理内存/RAM。
    4. 异步：由于资源有限，程序/进程的执行是走走停停而不是一贯到底。
  • 运行机制：
    1. 两种指令：特权指令与非特权指令
    2. 两种处理器状态：核心态与用户态
    3. 两种程序：内核程序与应用程序
  • 操作系统的结构：
    1. 内核：(必不可少的功能)
      • 时钟管理、中断处理、设备驱动、cpu切换、原语(原子操作，通过关中断和开中断来保证在开中断之前程序不可以被打断)
      • 四种主要功能(进程管理、内存管理、文件管理、设备管理)
    2. 非内核：
      • 非内核功能

二.OS框架

  • 由中向上(由OS到应用程序和用户)：
    1. 作用：向用户提供服务
    2. 实现方式：
      • 命令接口：终端输入指令
      • 程序接口：系统调用
      • GUI：图形界面
  • 由中向下(由OS到硬件)：
    1. 作用：更好的管理资源
    2. 实现方式：
      • 进程管理
      • 内存管理
      • 文件管理
      • 设备管理(磁盘属于内部设备)

三.中断、异常以及系统调用

  • 中断(外中断)简介：
    1. 来源：来源于外设(cpu外部，如不同硬件设备的计时器和网络的中断)
    2. 概念：通过中断使操作系统来感知到外设发出了请求，并对其进行处理，cpu的状态从用户态转为核心态。(注：内外中断是用户态切换为核心态的唯一途径)
    3. 处理时间：异步
    4. 处理响应的状态：持续，因为中断时操作系统会悄悄地完成接下来的工作，而应用程序不会感觉到中断。
  • 异常(内中断)简介：
    1. 来源：来源于不良的应用程序或者应用程序意想不到的行为(cpu内部)
    2. 概念：不良的应用程序发出的非法指令或其他坏的处理状态(如内存出错)，使得操作系统不得不完成相应的功能。
    3. 处理时间：同步
    4. 处理响应的状态：操作系统会杀死该程序或者重新执行该意想不到的应用程序指令。
  • 系统调用(内中断)简介：
    1. 来源：来源于应用程序内部
    2. 概念：是应用程序主动向操作系统发出的服务请求，并希望操作系统提供相应的支持，只要是与资源有关的操作都需要进行系统调用来保证系统的安全性。
    3. 处理时间：同步或异步(应用程序发出调用请求时，应用程序和操作系统是同步的，而操作系统返回调用时有可能与应用程序异步)
    4. 处理响应的状态：等待和持续
  • 中断的处理机制：
    1. 硬件需要做的处理：(中断的来源是外设以及硬件)
      • 将所有事件设置中断标记，一旦产生中断，将该标记提供给操作系统，操作系统根据标记也就知道了何处出现了问题。
    2. 操作系统需要做的处理：
      • 保存当前处理状态，即中断前一时刻的工作状态。
      • 根据中断标记，处理该中断程序。
      • 清楚中断标记。
      • 恢复之前的处理状态。
  • 异常的处理机制：(类似于硬件的中断标记，异常也有一个异常编号)
    1. 操作系统需要做的处理：
      • 保存现场
      • 异常处理(根据异常编号，杀死异常或者重新执行异常指令)
      • 恢复现场
  • 系统调用的处理机制：
    1. 应用程序进行系统调用时，是通过访问高层次的API接口而不是直接进行系统调用，操作系统再根据具体的调用对接硬件资源。

四.进程管理

  • 定义：
    1. 进程实体：由PCB(process control block–进程存在的唯一方式，保存了进程的状态等信息，用于恢复进程)、程序段、数据段组成，运行时在RAM中产生。
    2. 进程：进程是进程实体的一次运行过程，是资源分配的一个基本单位。
  • 组织方式：
    1. 链接方式：按照进程状态将PCB存储为队列形式，操作系统持有指向队列的指针。
    2. 索引方式：按照进程的状态建立索引表，操作系统持有指向索引表的指针。
  • 特征：
    1. 动态性：产生到消亡
    2. 并发性：可并发执行
    3. 独立性：独立运行
    4. 异步性：走走停停
    5. 结构性：每个进程都有一个PCB、程序段、数据段
  • 进程的状态以及状态转换：
    1. 状态：
      • 创建状态：操作系统为进程分配内存空间等资源，并为其创建以及初始化PCB的状态。
      • 就绪状态：已经拥有来运行所需要的资源，等待cpu空闲即可运行的状态。
      • 运行状态：占有cpu，并且正在cpu上运行的状态。
      • 阻塞状态：等待某个资源或者等待某个事件因而暂时不能运行的状态。
      • 终止状态：进程结束或者撤销并对资源进行回收的状态。
      • 挂起状态：引入虚拟内存技术，将暂时不能运行的进程调至外存等待，但该进程的PCB不会调到外存。
    2. 状态之间的转换：
      • 上述状态之间的转换。
  • 进程通信：
    1. 定义：进程之间的信息交换
    2. 方式：共享存储、管道通信、消息传递
  • 线程：
    1. 简介：程序执行流和cpu调度的最小单位，共享进程的资源。
    2. 属性：线程ID、TCB(Thread control block)、执行状态
    3. 实现方式：
      • 用户级线程：线程由应用程序通过线程库实现，cpu在用户态下即可完成，即用户能看到的线程。
      • 内核级线程：在核心态下完成的线程，即操作系统内核能看见的线程，内核级线程才是cpu分配的单位。
  • 进程调度的方式：
    1. 非抢占式调度：只允许进程主动放弃cpu。
    2. 抢占式调度：可以强制进程放弃cpu。
  • 调度算法：
    1. 先来先服务(FCFS)：先来先被调度
    2. 最短时间优先(SJF/SPF)：当前已到达且运行时间最短的先被调度
    3. 最高响应比优先(HRRN)：当前响应比最高的先被调度，其中响应比为：(等待时间+要求运行时间)/要求运行时间
    4. 时间片轮转调度算法(RR)：按照进程到达就绪队列顺序，轮流分配一个时间片，若进程在时间片内未完成，则剥夺cpu重新排队。
    5. 优先级调度算法：当前优先级最高的进程先被调度
    6. 多级反馈队列调度算法：
      • 是以上算法的平衡
        先设置多级优先队列，优先级从高到低，时间片从小到大。
        新进程到达时进入优先级最高的第一级队列，按FCFS原则等待分配时间片。
        时间片内未结束，则该进程进入下一个优先级队列。
        只有第k级队列为空时，才为第k+1级队列分配时间片。
  • 进程同步以及进程互斥：
    1. 简介：同步指的是进程之间的切换与执行满足一定的顺序，而异步指的是由于并发性，进程之间的执行顺序有多种交叉执行的可能，没有固定的顺序。
    2. 进程同步：亦称直接制约关系，指某些产生直接合作的进程为了完成某种任务协调他们的工作次序而产生的制约关系。
    3. 进程互斥：亦称间接制约关系，指一个进程访问某个临界资源(比如打印机)时，另一个进程必须等待。
    4. 用来实现同步和互斥的信号量机制：
      • 信号量：某种资源的数量
      • P操作/wait操作：根据当前信号量的数量判断能否需要等待/能否使用该资源，若能使用，则该资源的信号量减一。
      • V操作/signal操作：唤醒一个等待队列的进程/释放该资源，且该资源的信号量加一。
      • 同步/互斥信号量S：初始为零，用来配合P操作不能提前执行。
      • 同步的实现：先V后P，在进程一结束时执行V操作释放资源，进程二开始时执行P操作根据进程一是否完成决定能否进入，从而完成同步。同步的进程PV操作分开执行。
      • 互斥的实现：先P后V，进程开始减一使用资源，进程结束加一释放该资源。互斥的所有进程都需要执行P、V操作。
    5. 管程：
      • 意义：更容易的实现进程之间的同步与互斥。
      • 概念：将P、V操作封装，只允许一个进程在管程内执行。在Java中的关键字是synchronized。
  • 死锁：
    1. 定义：各个进程等待对方的资源，导致进程发生阻塞无法推进。
    2. 死锁的四个必要条件：
      • 互斥使用资源
      • 不剥夺/非抢占式
      • 占有一个资源且请求另一个资源
      • 产生循环等待链
    3. 如何避免死锁：
      • 破坏上述四个条件之一
      • 银行家算法：系统分配资源之前预先判断这次分配会不会导致系统进入不安全状态，以此决定是否执行这次分配。(是否处于安全态在于能否找到一个安全序列)
    4. 死锁检测算法：按照资源分布图，依次消除与非阻塞进程相连的所有的边，直到无边可消。

五.内存管理

  • 基础：
    1. 内存的意义：
      • 只有被放到RAM/内存中的程序才会被cpu处理。
    2. 内存的层次结构：
      • 从cpu寄存器到cache缓存、到主存、到磁盘，执行速度从快到慢。
    3. 操作系统需要分配的两块空间–逻辑空间与物理空间：
      • 逻辑地址空间：此空间是一个虚拟空间，给应用程序提供一个逻辑地址(相对地址)，使其在不接触底层硬件的情况下进行连续的工作。
      • 物理地址空间：分为主存和磁盘。
        主存：给优先级较高的正在运行的应用程序提供内存空间。
        磁盘：给优先级较低或者正在等待的应用程序提供内存空间。
    4. 地址空间：
      • 地址空间定义：
        • 逻辑地址空间：一个运行程序所拥有的内存范围
        • 物理内存空间：硬件支持的地址空间，主存以及磁盘。
      • 地址空间如何生成以及逻辑地址如何转换为物理地址：
        • 逻辑地址空间：程序的每行源码经过编译器等过程生成一个逻辑地址(相对地址)，最终整个程序会生成一段连续的逻辑地址。
        • 逻辑地址(相对位置)转换为物理地址(绝对位置)的步骤：
          • 操作系统提供逻辑地址和物理地址之间的映射
          • cpu中的存储管理单元(MMU)以RAM为桥梁根据映射找到逻辑地址对应的物理地址
      • 地址安全检查/内存保护(操作系统如何保护地址空间，即如何使进程只能访问该进程的物理地址空间)：
        • 操作系统根据逻辑地址的基址和界限(相当于起始地址和长度)确认该地址是否合法，若不合法则会产生OutOfMemory异常，操作系统对其进行异常处理。
    5. 三种将进程装入内存/RAM的方式：
      • 绝对装入：编译时直接产生绝对地址，适用于无操作系统。
      • 静态重定位装入：在程序装入时将逻辑地址/相对地址转换为物理地址/绝对地址。
      • 动态重定位装入：在程序运行时动态地将逻辑地址/相对地址转换为物理地址/绝对地址。
  • 实现虚拟内存的三种技术：
    1. 覆盖技术：(在较小的可用内存中运行较大的程序，发生在程序之内)
      • 按照程序自身的逻辑结构，将程序分成若干个独立的程序模块，那些不会同时运行的模块共享同一块内存空间，这些程序模块按时间顺序执行。必要或常用的模块放入内存，不常用的放入外存。不存在调用关系的模块不必同时存入内存，从而完成了覆盖，且共享了同一块内存。
        总结：就是将RAM按程序的逻辑结构分层/分区(可以将其看成树结构)，分区原则就是每块区域中可能存放的程序模块不会同时运行/相互独立。
    2. 交换技术：(发生在程序之间)
      • 操作系统将需要运行的程序导入到内存，将不需要运行的程序导出到外存/硬盘(进程挂起)。(单位是程序而不是模块，区别于覆盖技术)
    3. 虚存技术(虚拟内存管理技术)：
      • 程序之内既采用覆盖技术的思想也采用部分交换技术的思想，操作系统结合非连续式分配内存的方式将要用的信息调入内存，将暂时不用的信息调出外存。
  • 内存管理：
    1. 连续内存分配策略：(连续是指为进程分配的是一个连续的物理内存空间)
      • 首次适配：从地址为零的地方开始，将第一块满足程序空间使用要求的可用空闲块分配给该程序。
      • 最优适配：将最小的满足程序空间使用要求的可用空闲块分配给该程序。
      • 最差适配：将最大的可用空闲块分配给该程序。
      • 以上三种内存分配方法前提是已经分配好固定的空闲块，且都会产生内存碎片，还需要进行碎片整理(压缩式碎片整理、交换式碎片整理)，以提高资源的利用率。
      • 动态分区分配：在进程装入内存时(还未建立分区，也就没有空闲块)，根据进程的大小动态的建立分区。
    2. 非连续内存分配管理：(非连续是指为进程分配的是一个离散的物理内存空间)
      • 分页管理：(配合动态重定位的装入方式，更好的实现逻辑地址向物理地址的转变)
        第一步：
        将内存空间分为一个个大小相等的小分区，每个分区就是一个页框/页帧，每个页框/页帧都有一个编号就叫做页框号/页帧号。
        第二步：
        将要进入内存的进程的逻辑地址空间分成与页框/页帧的大小相等的一个个区域，这些区域称为页/页面，每个页面也有一个编号叫做页号。
        第三步：
        操作系统以页框为单位为进程分配内存空间，进程的每个页面放入一个页框/页帧中(每个页面不必连续存放)。根据页表对应的关系可以查询到逻辑地址对应的物理地址。
        注：页表–为了分页后将逻辑地址转为物理地址，页表保存了每个页面在内存中存放的位置。
        注：TLB–cpu中的快表，用于将常用的页表项(可以看成存储逻辑地址和物理地址映射的一个数组)缓存到cpu中，缩短寻址时间。
      • 分段管理：(与分页类似，区别就是分段地址偏移是可变的，分页的页帧的大小是固定的，分段比分页更容易实现信息的共享与安全。)
        将逻辑地址连续的程序按照一定规则(比如按照堆、栈、共享数据等程序自身的逻辑关系)分段，按照映射将这些段不连续的插入到物理地址中，可以避免连续时产生的内存碎片问题。
        注：段表–类似与页表
      • 段页式管理：
        将进程按逻辑关系分段，再将各段分页。
    3. 页面置换算法以及策略：(当内存已满而需要调入新的页面时，需要选择对哪个页面进行置换)
      • 算法：
        1. 最佳置换算法
        2. 先进先出算法
        3. 最久未使用算法
        4. 时钟置换算法
      • 策略：
        1. 局部页面置换：当某进程发生缺页时(访问的资源挂在外存中)，只能选在RAM内该进程自己的其他物理块进行置换。
        2. 全局页面置换：当某进程发生缺页时，操作系统可以将空闲块分配给该进程或者将其他进程持有的物理块置换到外存从而空出内存中的物理块给该进程。

六.文件管理

  • 基础：
    1. 文件定义：一组有意义的/信息数据集合。
    2. 文件属性：文件名、标志符(操作系统用来区分文件的内部名称)、文件类型、位置(包括用户角度的存放路径以及操作系统角度的外存中的地址)等。
  • 文件之间的组织方式：按照树的形式有序组织的一层层目录结构。
  • 文件内部的组织方式(逻辑结构方面)：
    1. 无结构文件：如TXT文件，由一些二进制或字符流组成。
    2. 有结构文件：
      • 简介：如数据库表，由一组相似的记录组成，每条记录又由若干的数据项组成。
      • 逻辑结构：
        1. 顺序文件：文件的记录逻辑上按一定顺序排列，物理上可以是顺序存储也可以是链式存储。
        2. 索引文件：由索引表(定长记录的顺序文件)和该索引表中对应每条记录(逻辑文件)的索引项组成。
        3. 索引顺序文件：索引文件与顺序文件的结合。
  • 文件在外存中的管理方式(物理结构方面)：
    1. 对非空闲磁盘块的管理：
      • 连续分配：通过连续分配形成的结构，由于磁盘块的读取是通过移动磁头实现的，故连续结构移动磁头所需的时间最短，文件在此结构下读写(I/O)速度最快，缺点是拓展性差。
      • 链接分配：通过链接/离散分配形成的结构，故拓展性强，外存利用率高，但是查找效率低。其中采用隐式链接分配的不支持随机访问，但采用显式链接分配但支持随机访问。
      • 索引分配：通过索引分配形成的结构，由存放索引的索引表和存放数据的数据块组成，易拓展读写慢。
    2. 对空闲磁盘块的管理：
      • 空闲表法
      • 空闲链表法
      • 位示图法
      • 成组链表法
  • 文件系统提供给用户的功能：
    1. creat：用户点击该功能时会调用”creat系统调用”。
    2. delete：用户点击该功能时会调用”delete系统调用”。
    3. 读/read：用户点击该功能时会调用”read系统调用”。
    4. 写/write：用户点击该功能时会调用”write系统调用”。
    5. open：用户点击该功能时会调用”open系统调用”。
    6. close：用户点击该功能时会调用”close系统调用”。
  • 文件保护：
    1. 口令保护：通过保存在系统中的口令实现
    2. 加密保护：通过加密密码和解密密码实现
    3. 访问控制：通过访问控制列表记录各个用户的权限实现
  • 文件系统的层次结构(从上往下)：
    1. 用户/应用程序
    2. 用户接口：提供接口
    3. 文件目录系统：提供路径
    4. 存取控制模块：访问权限检查，保证安全
    5. 逻辑文件系统与文件信息缓冲区：将文件索引表中的记录号转为文件的逻辑地址
    6. 物理文件系统：将逻辑地址转为物理地址
    7. 辅助分配模块：分配与回收存储空间
    8. 设备管理模块：与硬件设备交互，如磁盘调度、启动设备、释放设备。
    9. 设备
  • 磁盘调度：
    1. 调度时间：
      • 寻找时间：将磁头移动到目标扇区所在的磁道所花的时间
      • 延迟时间：通过旋转磁盘使磁头定位到目标扇区的时间
      • 传输时间：通过磁头和磁盘的转动读写数据的过程所花的时间
    2. 调度算法：(延迟时间和传输时间由硬件决定故操作系统只能优化寻找时间)
      • 先来先服务(FCFS)：按进程的顺序调度
      • 最短寻找时间优先：贪心算法，每次选当前最优
      • 扫描算法：在最内磁道和最外磁道之间来回(类比电梯，与电梯不同的就是无论最内最外有没有需要的数据都必须到最内或者最外才会返回)
      • 循环扫描算法：磁头朝特定方向才会处理读取数据，返回时则快速返回不处理数据。

七.外部I/O设备管理

  • 简介：外部设备可以看成一种特殊的文件。
  • 分类：
    1. 人机交互类：鼠标、键盘、打印机等
    2. 外部存储设备：移动硬盘、光盘等
    3. 网络通信设备：路由器，调制解调器等
  • I/O控制器(在外接设备的内部)：
    1. 意义：
      • cpu控制外部I/O设备的工具，相当于cpu与外部I/O设备之间的”中介”。
    2. 组成：
      • 控制器与cpu之间的接口
      • I/O逻辑(用来直接与cpu通信的模块)
      • 控制器与设备之间的接口
    3. 控制器与cpu之间的接口/功能：
      • 接收和识别cpu发出的命令：通过I/O控制器里的控制寄存器存放的命令和参数来识别。
      • 向cpu报告设备的状态：通过I/O控制器里的状态寄存器中的状态记录输出给cpu。
      • 数据交换：通过I/O控制器里的数据寄存器来寄存cpu(设备)发来的数据然后发送给设备(cpu)。
    4. I/O逻辑：
      • 地址识别：I/O控制器中用来区别上述三种寄存器的地址，根据地址判断cpu需要实现哪种功能。
    5. 控制器与设备之间的接口/功能：
      • 由一系列控制器与设备之间的接口来实现。
  • I/O软件层次结构：(由上往下)
    1. 用户层软件
    2. 设备独立性软件
    3. 设备驱动程序
    4. 中断处理程序
    5. 硬件
  • 脱机：脱离主机的控制进行输入/输出，例如提前将数据输入到磁带。
  • 假脱机技术(在用户层软件中实现)：用软件来实现脱机的技术。
  • 设备的分配与回收(设备独立性软件中实现)：
    1. 分配方式：
      • 安全分配方式：为进程分配一个设备后将该进程阻塞，直至I/O完成才会唤醒该进程。
      • 不安全分配方式：为进程分配一个设备后不对该进程阻塞。
      • 静态分配：进程运行前为其分配全部所需资源。
      • 动态分配：进程运行时动态的为其分配资源。
    2. 回收：略
  • 缓冲区：
    1. 定义：
      • 缓冲区是一块存储区域，可以用硬件寄存器，也可以用内存作为缓冲区。
    2. 作用：
      • 缓冲cpu和I/O设备之间速度不匹配的矛盾，减少cpu的中断频率。
    3. 缓冲区管理：
      • 单缓冲：只有一个缓冲区，缓冲区非空时不能向该缓冲区冲入数据，即缓冲区为空时才能冲入数据，缓冲区为满时才能传出数据。
      • 双缓冲：由两个缓冲区组成。
      • 缓冲池：由一系列的缓冲区组成。
